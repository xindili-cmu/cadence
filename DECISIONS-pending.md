## 原始三项(存档)

## 1. 分数区分度(动 rubric)

**问题.** SIGNAL 分挤在 70–75(约七成),85 封顶,90+ 恒空。分数越像常数,读者越不信它在"打分";三档「强信号 / 值得关注 / 参考」区分不出实质差异。

**约束.** 分数是 pinned 的(同一篇复用首次得分,防 85↔90 churn)。重调 rubric → 新旧文章标准不一致、**分数不可比**,展示档位(85/75/65)也要跟着重定。

**选项.**

- [ ] A. 拉开档距,让高分真稀缺(改 rubric + 洗存量分 + 重定展示档)—— 最彻底,代价最大
- [ ] B. rubric 不动,只重调**展示阈值**(比如 82/72/62),让三档人数更均衡 —— 轻,不碰存量分
- [ ] C. 不动,接受分数扁平

**建议.** B。先用展示层解决"三档挤在一起",不动评分基准就没有新旧不可比问题;真觉得不够再上 A。

---

## 2. 单刊浓度

**问题.** Sports Medicine 一家占 feed ~19%,读起来像"Sports Med 转发器",削弱"多刊全扫描"的卖点。

**约束.** 这是白名单抓取量的结构性结果。治理 = 主动丢弃部分高分文章。

**选项.**

- [ ] A. 单刊单日配额(如 ≤N 条),超出的降到快讯 / 次日 —— 治本,但会丢内容
- [ ] B. 只在 UI 上做"同刊折叠"(展示层限流,数据不丢)—— 轻,治标
- [ ] C. 不动,认为高产刊占比高本就合理

**建议.** 先看真实分布再定。如果确实是一刊长期霸榜 → A(配额设宽一点,比如单刊单日 ≤5);如果只是偶发 → C。**要不要我先出一份近 30 天各刊占比?**

---

## 3. 收藏 / 个性化(工具化)

**问题.** 北极星是"PT 绕不开的**工具**",但现状是**内容流**——无收藏、无按专科订阅、无"我的"状态。回访全靠"今天更新了"。工具留资产,内容流不留。

**约束.** 这是加功能不是修 bug,方向有轻重之分(要不要登录 / 本地存储 / 做多深)。

**选项.**

- [ ] A. 纯本地收藏(localStorage,无需登录)+ 卡片加星标,单开"我的收藏"视图 —— 副作用可控,不动后端
- [ ] B. A + 按专科的 email 订阅分段(选专科,只收该专科周报)—— 中等,动订阅管线
- [ ] C. 账号体系(登录 / 跨设备同步)—— 重,先不碰
- [ ] D. 不动

**建议.** A 先上(最轻的"工具化"信号,零后端);跑一阵看有没有人用,再决定要不要 B。
