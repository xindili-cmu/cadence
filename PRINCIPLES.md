## 1. 目标 (Goal)

**主方向是 builder:把 Cadence 做成 PT 临床绕不开的工具——让治疗师"跟上证据"这件事离不开它。**

"能养活自己"是这个目标的**底线约束**,不是另一个方向:

- 它是**跑道**——有收入,才能持续把工具做下去,不至于中途饿死。
- 它是**证据**——PT 愿意为 Cadence 付费,本身就是"它正在变必装"的最强信号。

**冲突时:** 短期先保跑道(不死);但绝不用让工具"更不必装"的方式赚钱——不牺牲筛选质量、不在产品被真正喜欢之前就把核心价值锁进付费墙、不为短期现金做偏离产品的事。

**量化锚点(3 个月内):** 每篇推送阅读人数 ≥ 粉丝数的 50%,且每周至少有 1 次来自 PT 的主动转发。

---

## 2. 核心原则 (Principles)

- **人来收尾,绝不 slop。** AI 出稿,最后 10% 必须人工把关——这是底线,即使某条 slop 会爆也不做。
- **去 AI 感。** 刊头忌 `VOL.` / 全大写 mono 风;篇数放日期行。文案用「我们帮你」,不用「AI 帮你」。
- **内容优先,不做站内漏斗。** 日报默认**去导流、可助推**;不放 CTA、不放阅读原文外链(2026-06-26 决定,已放弃站内漏斗)。
- **节奏固定。** 5×/周:周二–周六发(覆盖美国周一–周五),周日 / 周一停更。
- **发布永远由人。** 所有平台一律停在发布键,Cindy 亲自点发。
- **变现不反噬产品。** 短期保跑道,但绝不靠"让工具更不必装"的方式赚钱:不牺牲筛选质量、不在被真正喜欢前锁付费墙、不为短期现金偏离产品。
- **keep the main thing the main thing。** 主线是把日报 + 网站做好;会议 / 约稿 / 旁支默认说不。

---

## 3. 定位 (Positioning)

- **品类:** 康复(rehab / PT)高分文献策展。
- **渠道:** 公众号「Cadence 步频」/ 小红书同名 / LinkedIn + 自有网站(含信号分滑块筛选 `?min=`)。
- **策展引擎:** 生产环境用 `gemini-2.5-flash` 打分(repo `LLM_PROVIDER=gemini`,仅 `GEMINI_API_KEY` 配齐),评分字段 `curatedScore`。
- **SEO:** GSC 域名已验证、sitemap 已提交;Googlebot 能完整渲染 React,**无需 SSR**。

---

## 4. 质量门槛 (Quality bar)

- **信号分档:** 85 / 75 / 65 → 对外表述「强信号 / 值得关注 / 参考」(2026-07-04 与站内 UI 对齐——实际评分在 85 附近封顶,90 档永远空置,故弃用旧 90/80/65 档;见 components.bundle.jsx 注释)。
- **相关性门:** HARD denylist(硬药物)/ SOFT(疾病);theme leg 仅供研究,不外发。
- **公众号标题:** 固定格式「步频日报丨{M.D} 高分康复文献速读」(标题栏 + 封面);摘要为内容型,每天不同、不复用。
- **每期呈现公众号 brief 时,附当天 `meta.txt` 的标题 + 摘要(可复制),逐日不同。**

---

## 5. 运营护栏 (Operational guardrails)

- **不在 Linux 沙箱跑 `npm run build-app`**(共享 node_modules,esbuild 平台二进制会坏 Mac 本地);`app.min.js` 由 Cindy 本地重建。
- **沙箱连不上 LLM API** → backfill 类脚本只在 Cindy 终端跑。
- **仓库更新后只输出 bash add/commit/push 块**,Cindy 自己终端执行;每日 cron 推 ~9 提交常致本地 push 被拒,用 `git pull --no-rebase` 干净合并。
- **公众号粘贴用 `<section>` 不用 `<div>`**(背景会被剥)。
- **发布 runbook:** `PUBLISHING.md`(XHS 半自动整段注入避免中文损坏;公众号纯手动)。

---

## 6. 什么给 / 耗能量 (Energy)

- **给能量:** vibe coding(写 / 改脚本与网站)、视觉设计(卡片 / 封面 / 刊头)、写作打磨。
- **耗能量:** 跨平台手动复制粘贴、发布踩坑调试(中文损坏 / 样式被剥 / push 冲突)。

> 推论:**耗能量的两件事正是最该自动化的**——它们既不产出价值,又消耗你做"给能量"那三件事的精力。
